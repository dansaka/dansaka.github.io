被你忽视的 AS模版



被你忽视的 AS模版

在开发我们利用as的模版功能，定义一些开发中常用的代码模版可以提升我们的开发效率。下面我们就来看下如何使用as的模版：

File Header

每当我们new 一个类的时候，可能 需要添加 作者 日期 描述文案。

/**
 *  @author : ${USER}
 *  @date : ${DATE} ${TIME}
 *  description : 
 */

入口如下图：

添加好了上面的File Header 当我们再次新建一个类的时候会自动带出：作者 日期 描述 当然你也可自定义其他你需要给这个类添加的统一头部信息。

Live Templates

As中已经为我们的定义了大量的模版 淡然我们不需要记住全部，记住几个常用的就行了，入口见下图：

下面我们例举几个常用的：

foreach

for ($i$ : $data$) {
    $cursor$
}

gone

$VIEW$.setVisibility(android.view.View.GONE);

logd

android.util.Log.d(TAG, "$METHOD_NAME$: $content$");

lwm

android:layout_width="match_parent"

fori

for(int $INDEX$ = 0; $INDEX$ < $LIMIT$; $INDEX$++) {
  $END$
}

官方提供我们可以按需选择，最有用的可以按自己的习惯去定义一些常用代码块/布局块。。。

上面都是小儿科 重头戏来了

在new的时候都见过上图中的红框中的选项，他们是as自定好的一些列强大的模块。我们在输入一些信息关联的生成我们开发中所需的一系列文件和类。一般可能用不到as自定义好的模版。需要我们按照自己的需求定义自己的模版。

那么怎么去自定义自己的模版呢：我们需要参照官方定义好的自己定义模版文件，来修改符合自己的要求的模版：

官方的模版文件在：/Applications/Android\ Studio.app/Contents/plugins/android/lib/templates

上图的WMS Activity 就是我自己的安装官方文件修改后自定义的一个模版 一键生成activity xml 已经把activity注册进manifest 。当然可以根据自己的需求生成更多的文件和类。比方说MVP架构，需要建的的类文件很多，这个时候自定义模版一键生成相关的类是非常方便。

对照官方的模版文件。相信都能自定义出自己的模版文件。需要注意一点就是新建完成放在/Applications/Android\ Studio.app/Contents/plugins/android/lib/templates文件夹下 ，需要重启AS才能生效。动手试试吧。如遇到问题可以与我联系。
